# LH 신도시 교통 인프라 데이터 분석을 통한 교통약자 안전 대안 도출

> **팀명:** 천천히, 꾸준히  
> **팀원:** 고은성, 전창현, 최정인

---

## 1. 과제 개요

본 보고서는 4개 신도시(동탄1·2, 위례, 하남미사, 판교)의 교통사고·교통량·인프라·인구 데이터를 통합 분석하여, 교통약자(어린이·노인 등) 중심의 위험지역을 도출하고 하남교산 적용 대안을 제시하는 것을 목적으로 한다.

핵심 질문은 다음과 같다.
- 어떤 격자/도로가 실제로 위험한가?
- 위험의 원인은 무엇인가?
- 제한된 예산에서 어떤 안전시설을 어디에 우선 배치해야 하는가?

### 2-2. 정제/가공 절차

분석은 `link_id`(도로 단위)와 `grid_gid`(100m 격자 단위)를 병행하여 수행했다.

**1) 사고 심각도 표준화(EPDO)**

> **EPDO란?** Equivalent Property Damage Only의 약자로, 사고의 심각도(사망·중상·경상 등)를 하나의 숫자로 환산한 지표이다. 예를 들어 사망사고 1건은 경미한 물적피해 391건과 같은 수준의 위험으로 본다.

$$
\text{EPDO}\_{i} = \sum\_{k} (w\_{k} \times n\_{ik})
$$

- $n\_{ik}$ : 구간 i에서 발생한 심각도 k 사고 건수
- $w\_{k}$ : 사고 심각도에 대한 가중치
- $k \in$ {사망, 중상, 경상, 부상신고, 상해없음, 기타불명}

**EPDO 가중치:**

| 심각도 | 가중치 | 비고 |
|--------|--------|------|
| 사망 | 391 | PDO 391건 상당 |
| 중상 | 69 | PDO 69건 상당 |
| 경상 | 8 | PDO 8건 상당 |
| 부상신고 | 6 | |
| 상해없음(PDO) | 1 | 기준값 |
| 기타불명 | 8 | 경상과 동일, 보수적 처리 |

**가중치 산출 근거:**
> 이상엽(2019). 「교통사고 원인행위의 벌점 추정에 관한 연구」, 대한교통학회지, 37(5), 365-378.
>
> 해당 연구는 사고 심각도별 사회적 비용을 물적피해(PDO) 1건 대비 상대 비율로 환산한 것이다. 즉, 사망 1건의 사회적 비용은 물적피해 391건에 해당하며, 중상 1건은 물적피해 69건, 경상 1건은 물적피해 8건에 해당한다. 이를 통해 사고 건수 단순 합산의 편향을 보정하고, 심각도를 반영한 단일 지표(EPDO)로 통일했다.

**2) 도로(링크) 단위 위험도 산출**
- 도로의 노출량(= 연평균 일교통량 × 도로 길이)을 고려하여, 교통량 대비 사고 심각도 비율(`epdo_rate`)을 산출했다.
- 이를 통해 "교통량이 많아서 사고 건수가 많아 보이는 도로"와 "진짜 위험한 도로"를 구분했다.

**3) 격자 단위 통합**
- 사고, 인프라(14~21), 인구(03~07), 속도/혼잡(09/11/12) 데이터를 100m 격자(`grid_gid`) 기준으로 하나로 합쳤다.
- 값이 비어 있는 경우(결측)에는 0 또는 해당 변수의 평균값으로 채워 넣었다.

**4) 공간 처리 (지도 위에서 데이터 연결)**
- 사고 발생 지점(점 데이터)은 해당 위치가 속하는 격자 칸에 배정했다.
- 도로(선 데이터)는 격자 칸과 겹치는지 여부로 연결하여, 데이터 누락을 최소화했다.

**5) 품질 점검**
- 데이터가 정상적으로 연결되었는지(매칭률), 빠진 값이 얼마나 되는지(결측률)를 확인했다.
- 값의 범위가 비정상적이지 않은지(음수 발생, 극단적 이상값 등)를 점검했다.

### 2-2-1. 파생 변수 생성 과정

본 연구에서는 제공받은 원시 데이터를 가공하여, 교통약자 위험도를 설명할 수 있는 7개의 새로운 지표(파생 변수)를 만들었다.

> **격자(grid)란?** 분석 대상 지역을 100m × 100m 정사각형으로 나눈 단위이다. 각 격자마다 사고·인구·인프라 정보를 집계하여 비교한다.

**(1) 노인 거주 비율 (`elderly_res_ratio`)**

`03_성연령별_거주인구(격자).csv`에서 격자별 60대 이상 거주인구와 전체(20대 이상) 거주인구를 합산하여 비율을 산출하였다.

$$
\text{elderly\\_res\\_ratio} = \frac{\text{elderly\\_res\\_pop}}{\text{total\\_res\\_pop}}
$$

- elderly_res_pop = 60대~100대 이상 남녀 합산 (m_60g_pop + w_60g_pop + … + m_100g_pop + w_100g_pop)
- total_res_pop = 20대~100대 이상 남녀 합산

**(2) 교통약자 유동 비율 (`vuln_float_ratio`)**

`04_성연령별_유동인구.csv`의 위치 좌표(경도, 위도)가 어느 격자 칸에 속하는지를 판별하여, 격자별 어린이(10대)·고령자(60대 이상) 유동인구 비율을 산출하였다.

$$
\text{vuln\\_float\\_ratio} = \frac{\text{child\\_float} + \text{elderly\\_float}}{\text{total\\_float}}
$$

- child_float = m_10g_pop + w_10g_pop
- elderly_float = m_60g_pop + w_60g_pop
- total_float = 10대~60대 이상 전체 남녀 합산 (12개 컬럼)

**(3) 취약시간대 교통약자 인구 (`vuln_peak_pop`)**

어린이·고령자가 도로에 노출되는 시간대를 **등교(07–09시)**, **하교(13–16시)**, **퇴근(17–19시)** 총 10개 시간대로 정의하고, `05_시간대별_직장인구.csv`와 `06_시간대별_방문인구.csv`에서 해당 시간대 인구를 격자별로 합산하였다.

$$
\text{vuln\\_peak\\_pop} = \text{vuln\\_work} + \text{vuln\\_visit}
$$

- vuln_work = Σ TMST_{h} (직장인구), h ∈ {07, 08, 09, 13, 14, 15, 16, 17, 18, 19}
- vuln_visit = Σ TMST_{h} (방문인구), 동일 시간대

직장·방문인구 데이터는 수백만 행에 달하므로, 3만 행씩 나누어(청크 처리) 각 사람의 좌표가 속하는 격자를 찾아 합산하였다.

**(4) 주말 방문 비율 (`weekend_ratio`)**

`07_주중주말_서비스인구.csv`를 `hw` 컬럼 기준으로 주중(H)/주말로 분리한 뒤, 방문인구(`v_pop`)를 격자별 합산하여 주말 비율을 산출하였다.

$$
\text{weekend\\_ratio} = \frac{\text{weekend\\_visit}}{\text{weekday\\_visit} + \text{weekend\\_visit}}
$$

**(5) 격자 평균 속도 (`grid_avg_speed`)**

`08_상세도로망_네트워크.geojson`에서 각 도로 구간이 어떤 격자 칸을 지나가는지를 판별한 뒤, 격자 안을 통과하는 모든 도로의 평균속도를 평균 내어 해당 격자의 대표 속도로 사용하였다.

$$
\text{grid\\_avg\\_speed} = \frac{1}{|L\_{i}|} \sum\_{l \in L\_{i}} \text{avg\\_speed}\_{l}
$$

- $L\_{i}$ : 격자 $i$와 교차하는 도로 링크 집합 (avg\_speed > 0인 링크만)

속도 측정 기록이 없는 격자는, 해당 격자를 지나는 도로의 등급(고속도로/국도/지방도 등)에 따른 전체 평균속도를 대신 사용하였다.

**(6) 격자 혼잡위험도 (`grid_congestion`)**

각 도로 링크의 혼잡위험도는 11·12번 혼잡 데이터(FRIN_CG, TI_CG)를 이용하여 다음과 같이 산출하였다.

$$
\text{congestion\\_risk}\_{l} = \frac{\text{FRIN\\_CG}\_{l} \times \text{TI\\_CG}\_{l}}{100}
$$

- FRIN\_CG: 혼잡이 얼마나 자주 발생하는지 (0–100)
- TI\_CG: 혼잡이 얼마나 오래 지속되는지 (0–100)

이를 격자 안을 지나는 모든 도로에 대해 평균을 구하였다.

$$
\text{grid\\_congestion} = \frac{1}{|L\_{i}|} \sum\_{l \in L\_{i}} \text{congestion\\_risk}\_{l}
$$

**(7) 인프라 공백 수 (`gap_cnt`)**

아래 **2-3-1절**에서 별도로 정의한다.

### 2-3. 인프라 변수 정의(데이터 14~21)

| 변수명 | 설명 | 데이터 번호 |
|--------|------|-------------|
| child_zone_cnt | 어린이보호구역 수 | 14 |
| school_cnt | 학교 수 | 15 |
| kindergarten_cnt | 유치원 수 | 16 |
| daycare_cnt | 어린이집 수 | 17 |
| crosswalk_cnt | 횡단보도 수 | 18 |
| bus_stop_cnt | 버스정류장 수 | 19 |
| cctv_cnt | CCTV 카메라 수 | 20 |
| speedbump_cnt | 과속방지턱 수 | 21 |

해석: 각 격자 안에 안전시설이 얼마나 갖추어져 있는지를 나타내는 지표로 사용하였다.

#### 2-3-1. 인프라 공백 지표 정의 (`gap_cnt`)

안전시설이 얼마나 부족한지를 숫자로 표현하기 위해, **안전시설 6종**을 대상으로 인프라 공백 지표를 정의하였다.

**대상 시설(안전시설):**

| 변수명 | 시설명 |
|--------|--------|
| crosswalk_cnt | 횡단보도 |
| child_zone_cnt | 어린이보호구역 |
| speedbump_cnt | 과속방지턱 |
| cctv_cnt / cctv_cam_cnt | CCTV |
| bus_stop_cnt | 버스정류장 |

> 교육시설(school_cnt, kindergarten_cnt, daycare_cnt)은 안전시설이 아닌 **위험 유발 시설**(어린이 통행 발생원)로 분류하여 공백 산정에서 제외하였다.

**판정 기준:**
- 시설 수 = 0 → **없음** (공백)
- 시설 수 < 전체 격자 평균의 50% → **부족** (공백)
- 그 외 → 정상

$$
\text{gap\\_cnt}\_{i} = \sum\_{k=1}^{6} \mathbb{1}(\text{시설}\_{k} \text{가 공백})
$$

$$
\mathbb{1}(x) = \begin{cases} 1 & \text{시설 없음 또는 부족} \\ 0 & \text{시설 존재} \end{cases}
$$

즉, 해당 격자에 안전시설이 존재하지 않거나 현저히 부족한 경우 공백으로 계산하며, gap_cnt의 범위는 0~6이다. 이 지표는 이후 **종합 위험지수의 보정계수**(correction_index)와 **시설 제안 로직**에 직접 활용된다.

## 4. 최종 결과 도출 과정

### 4-1. 모델링 프레임

최종 위험도는 **세 가지 서로 다른 통계 기법**의 결과를 합쳐서(3중 통합 가중치) 산출했다. 하나의 방법에만 의존하면 편향이 생길 수 있으므로, 세 방법의 장점을 고루 반영하기 위함이다.

- `w_nb` — **음이항 회귀**(사고 건수를 예측하는 통계 모델)가 판단한 각 변수의 중요도
- `w_corr` — 각 변수와 **사고 심각도(EPDO)의 상관관계** 크기
- `w_entropy` — 격자 간 **값의 차이가 큰 정도**(엔트로피 방식)로 측정한 변수의 구분력

세 가중치를 **기하평균**(곱한 뒤 세제곱근)으로 합산한 후, 합이 100%가 되도록 정규화한다.

$$
w\_{\text{raw},i} = (w\_{\text{nb},i} \times w\_{\text{corr},i} \times w\_{\text{entropy},i})^{1/3}
$$

$$
w\_{\text{final},i} = \frac{w\_{\text{raw},i}}{\sum\_{j} w\_{\text{raw},j}}
$$

#### 4-1-1. 음이항 회귀 모델 선택 이유

교통사고 건수는 0, 1, 2, 3… 과 같이 **정수로만 나타나는 데이터**이다. 이런 데이터에는 보통 포아송(Poisson) 회귀가 쓰이는데, 포아송 모델은 "데이터의 평균과 흩어진 정도(분산)가 비슷하다"는 가정이 있다.

그런데 실제 데이터를 보면, 대부분의 격자는 사고가 적고 일부 격자에만 사고가 집중되어 **데이터가 평균보다 훨씬 넓게 퍼져 있는 현상(과산포, over-dispersion)** 이 나타났다.

| 통계량 | 값 |
|--------|------|
| 격자 수 (N) | 1,481 |
| 사고 건수 평균 | 2.91 |
| 사고 건수 분산 | 7.93 |
| **분산/평균 비** | **2.73** |

분산(7.93)이 평균(2.91)의 약 2.7배로, 포아송 모델의 "평균 ≈ 분산" 가정에 맞지 않는다. 따라서 이런 과산포 상황에 적합한 **음이항 회귀(Negative Binomial Regression)** 를 적용하였다.

> **음이항 회귀란?** 포아송 모델을 확장한 것으로, "눈에 보이지 않는 격자 간 차이"까지 고려하여 사고 건수를 더 정확하게 설명할 수 있는 통계 모델이다.

$$
\text{Var}(Y) = \mu + \alpha \mu^{2}
$$

- $\mu$ : 조건부 평균 (포아송 부분)
- $\alpha$ : 과산포 파라미터 ( $\alpha > 0$ 이면 과산포)

예측하려는 값(종속변수)은 격자별 사고 건수이며, 원인이 되는 변수(독립변수)는 `2-2-1절`에서 정의한 7개 파생 변수를 평균 0·표준편차 1로 맞춘 뒤(표준화) 투입하였다.

#### 4-2-1. 개별 가중치 산출 방법

**(1) 엔트로피 가중치 ( $w\_{\text{entropy}}$ )**

> **엔트로피란?** 데이터가 얼마나 고르게 분포되어 있는지를 측정하는 지표이다. 값이 격자마다 크게 다르면(변별력이 높으면) 가중치가 커진다.

격자 간 변동이 클수록 구분력이 높다는 원리.

$$
x'\_{ij} = \frac{x\_{ij} - \min}{\max - \min} \quad \text{(정규화)}
$$

$$
p\_{ij} = \frac{x'\_{ij}}{\sum\_{i} x'\_{ij}} \quad \text{(비율)}
$$

$$
e\_{j} = -\frac{1}{\ln n} \sum\_{i} (p\_{ij} \times \ln p\_{ij}) \quad \text{(엔트로피)}
$$

$$
w\_{j} = \frac{1 - e\_{j}}{\sum\_{j} (1 - e\_{j})} \quad \text{(가중치)}
$$

**(2) 음이항 회귀 가중치 ( $w\_{\text{nb}}$ )**

사고건수를 종속변수로 음이항 회귀 수행.

$$
X\_{j,\text{std}} = \frac{X\_{j} - \mu\_{j}}{\sigma\_{j}} \quad \text{(표준화)}
$$

$$
\ln(\mu) = \beta\_{0} + \sum\_{j} \beta\_{j} \times X\_{j,\text{std}} \quad \text{(모델)}
$$

$$
w\_{j} = \frac{|\beta\_{j}|}{\sum\_{j} |\beta\_{j}|} \quad \text{(가중치)}
$$

**(3) EPDO 상관계수 가중치 ( $w\_{\text{corr}}$ )**

각 변수가 사고 심각도(EPDO)와 얼마나 같이 움직이는지(상관관계)를 측정하여 가중치로 변환하였다.

$$
w\_{j} = \frac{|r\_{j}|}{\sum\_{j} |r\_{j}|}
$$

#### 4-2-2. 통합 (세 가중치를 하나로 합침)

$$
w\_{\text{raw},j} = (w\_{\text{nb},j} \times w\_{\text{corr},j} \times w\_{\text{entropy},j})^{1/3}
$$

$$
w\_{\text{final},j} = \frac{w\_{\text{raw},j}}{\sum\_{j} w\_{\text{raw},j}}
$$

예) `vuln_peak_pop`: 0.4035 / 0.7908 = 0.5103 = **51.03%**

### 4-3. 종합 위험지수 산출

격자별 최종 위험도는 `08_격자_종합위험지수.csv`의 종합지수(`entropy_composite_risk`)를 기준으로 순위를 부여했다.

#### 4-3-1. 종합 위험지수 계산 방식

종합 위험지수(`entropy_composite_risk`)는 **사고 심각도(EPDO)에 주변 환경 보정값을 더하는 방식**으로 산출하였다. 사고가 심각한 곳일수록 기본 점수가 높고, 노인 밀집·인프라 부족 등 환경이 나쁠수록 추가 점수가 붙는 구조이다.

**① 보정지수 산출 (`correction_index`)**

7개 파생 변수(2-2-1절 참조)를 모든 격자 중 최솟값=0, 최댓값=1이 되도록 **0~1 범위로 환산(min-max 정규화)** 한 뒤, 각 변수의 중요도(엔트로피 가중치)를 곱해서 합산한다.

$$
X'\_{ij} = \frac{X\_{ij} - \min(X\_{j})}{\max(X\_{j}) - \min(X\_{j})}
$$

$$
\text{correction\\_index}\_{i} = \sum\_{j=1}^{7} w\_{\text{entropy},j} \times X'\_{ij}
$$

| 인자 ( $j$ ) | 엔트로피 가중치 ( $w\_{\text{entropy},j}$ ) |
|---|---|
| elderly_res_ratio | 0.4753 (47.53%) |
| vuln_peak_pop | 0.3478 (34.78%) |
| grid_congestion | 0.0687 (6.87%) |
| grid_avg_speed | 0.0525 (5.25%) |
| weekend_ratio | 0.0249 (2.49%) |
| gap_cnt | 0.0246 (2.46%) |
| vuln_float_ratio | 0.0062 (0.62%) |

> 여기서 사용하는 가중치는 엔트로피 방법 **단독으로** 산출한 값이며, 4-2절의 "세 방법 통합 가중치"와는 다른 값이다. 통합 가중치는 변수 순위 결정에, 엔트로피 단독 가중치는 위험지수 보정에 각각 쓰인다.

**② 종합 위험지수 산출**

$$
\text{entropy\\_composite\\_risk}\_{i} = \text{EPDO}\_{i} \times (1 + \text{correction\\_index}\_{i})
$$

- EPDO가 높을수록(사고가 심각할수록) 기본 위험 점수가 높다
- 보정지수가 클수록(노인 밀집, 등하교 시간대 인구 많음, 안전시설 부족 등) 위험 점수가 추가로 올라간다
- 보정지수의 범위는 0~1이므로, 최대 2배까지 점수가 높아질 수 있다

**③ 산출 예시**

| 항목 | 값 |
|---|---|
| 격자 다사672368의 EPDO | 531.0 |
| elderly_res_ratio 정규화값 × 가중치 | 0.4753 × 0.67 = 0.318 |
| vuln_peak_pop 정규화값 × 가중치 | 0.3478 × 0.52 = 0.181 |
| (나머지 5개 인자 합산) | 0.109 |
| **correction_index** | **0.608** |
| **entropy_composite_risk** | 531.0 × (1 + 0.608) = **854.04** |

- `elderly_res_ratio`가 엔트로피 기준 핵심 변수로 확인되며, 고령자 보행 동선 중심 정책 설계가 필요하다.

## 8. 한계 및 후속 과제

- 본 분석은 "어디가 위험한가"를 순위로 보여주지만, "왜 위험한가"의 원인을 직접 증명하지는 못한다(상관관계 ≠ 인과관계).
- CCTV/방지턱은 **이미 위험한 곳에 사후에 설치**된 경우가 있어, "CCTV가 많을수록 사고가 많다"는 결과가 나올 수 있다. 이는 CCTV가 사고를 유발하는 것이 아니라 위험한 곳에 이미 설치되었기 때문이다.
- 1개 연도 데이터만 사용하였으므로, 여러 해에 걸친 검증과 시설 설치 전후 효과 비교가 필요하다.
