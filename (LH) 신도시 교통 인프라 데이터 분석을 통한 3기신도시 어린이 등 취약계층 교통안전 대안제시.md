# 신도시 교통 인프라 데이터 분석을 통한 3기신도시 어린이 등 취약계층 교통안전 대안제시

## 개요

### 분석배경

- 급격한 기후변화와 도시 인프라 노후화로 인해 전통적인 도시 관리 방식이 한계에 직면하고 있으며, 이를 극복하기 위한 데이터 기반의 새로운 도시 관리 대안이 요구되고 있습니다.
- 최근 어린이, 노인 등 교통약자의 교차로 우회전 사고율이 증가함에 따라, 이들의 안전을 확보하기 위한 교통안전 대책 마련의 필요성이 더욱 커지고 있습니다.
- 스마트시티 건설 과정에서 주민의 안전 취약 요인을 완화하고, 시민의 삶의 질을 향상시키기 위한 선제적이고 체계적인 대응 방안 수립이 필요한 상황입니다.

### 분석목적

- 도로상에서 발생하는 교통약자(어린이, 노인, 임산부 등) 대상 교통사고를 저감하기 위한 데이터 기반 교통안전 대책 도출을 목적으로 합니다.
  - 특히 초등학교 주변 스쿨존, 통학로, 교차로 우회전 구간 등에서 발생하는 어린이 교통사고 예방에 중점
  - 다양한 공간 데이터를 활용하여 사고 위험 발생 가능성을 진단
  - 분석 결과를 기반으로 실효성 있는 교통안전 대응 방안 수립에 활용

### 해결내용

- 분석대상지의 교통사고, 교통량, 유동인구 데이터를 활용하여 사고 유형별·연령별 교통사고 현황을 분석하고, 사고 발생 원인과 지역적 특성을 도출합니다.
- 분석 결과를 바탕으로 적용대상지의 토지이용계획상 도로 및 교차로 구조를 고려하여 스마트 안전 교통시설물의 적정 설치 위치를 예측합니다.
- 스마트 안전 교통시설물 설치에 따른 교통량 변화 및 사고율 변화 효과를 계량적으로 산출하고, 이를 반영하여 최적의 시설물 위치를 제시해야 합니다.
- 스쿨존, 통학로, 교차로 우회전 구간 등 교통안전 취약지역의 교통 여건을 반영한 맞춤형 대안을 제시해야 합니다.
  - 예시) 우회전 교차로 전광판 설치, 지능형 CCTV 설치, 가드펜스 설치 등
- 분석대상지 범위는 다음과 같습니다.
  - 분석대상지역 : 동탄 1·2 신도시, 위례 신도시, 하남미사, 판교신도시
  - 적용대상지역 : 하남교산

### 분석 시 고려사항

- 해결내용 중 과제에서 요구되는 결과 및 요인(지표) 도출과 해결방안을 제시해야 합니다.
- 현황 데이터 및 분석 결과는 Python, QGIS 등을 활용하여 시각화 결과를 설명해야 합니다.
- 분석 과정에서 논문, 통계자료, 공공데이터 등 외부 자료의 활용은 가능하나, 모든 참고 자료의 출처를 명확히 표기해야 합니다.
  - ※ 자체적으로 필요 데이터를 수집·발굴하여 사용 가능하나 최종 결과파일 제출시 활용 데이터 및 출처를 함께 제출해야 합니다.
  - ※ 외부 데이터는 법적인 제약이 없는 경우에만 허용되며 크롤링을 통해 데이터를 생산한 경우 해당 데이터와 크롤링 코드도 함께 제출해야 합니다.
  - ※ 최종 분석보고서 제출 시 발표용 프레젠테이션 스크립트 파일을 함께 제출해야 합니다.

- 최종 심사 결과 : 1차 평가 점수(40%)와 2차 평가 점수(60%)를 합산하여 선정

| | 최종 결과 도출을 위해 데이터 간 융·복합 분석이 적절히 이루어졌는가? |
| 분석의 창의성 | 문제 해결을 위해 창의적이고 차별화된 분석 접근을 시도하였는가? |

| 코드 완성도 | 제출된 코드가 오류 없이 실행 가능하며 재현성이 확보되었는가? |
| | 코드가 가독성 있게 정리되어 있으며 구조화가 적절히 이루어졌는가? |

## 결과 제출

### 제출 조건

- 소스 코드는 사용하는 패키지를 포함하여 오류 없이 실행 가능하도록 제출해야 합니다.
- 코드 및 주석 작성 시 문자 인코딩은 UTF-8을 사용해야 합니다.
- 공간 시각화 도구는 QGIS를 사용해야 합니다.
- 제출 파일의 공간정보(GIS) 좌표계는 WGS84 좌표계(EPSG:4326)를 사용해야 합니다.

### 제출 파일

- **(필수) 데이터 파일**
  - 전처리 결과 데이터, 분석 결과 데이터, 시각화 활용 데이터를 폴더별로 구분하여 정리
  - 구분된 데이터 파일은 하나의 ZIP 파일로 압축하여 제출
  - 제출 파일 형식 : SHP(Shape) 포맷 위치정보 파일, CSV 파일, 시각화 결과 파일
  - ※ 소스코드 실행 순서 관련 파일, 참고자료, 시각화 결과물, 사용 폰트 등은 함께 포함 가능

- **(필수) 소스 코드 파일**
  - COMPAS Notebook 소스코드 파일(*.ipynb 등) 제출
  - ※ 실행 순서를 명확히 구분한 파일명으로 제출 가능

- **(필수) 분석보고서 파일**
  - 데이터 정제·가공, 추가 수집, 알고리즘 도출 과정 및 결과를 설명한 분석보고서 제출
  - 보고서 첫 장에는 제목과 팀명만 기재하며 인적사항은 기재 불가

| 분석보고서 양식 | 분석보고서 필수 기재사항 |
|---|---|
| 자유 양식 | 데이터 전처리 및 분석 과정에 대한 상세 설명 |
| 분량 제한 없음 | 추가 수집 데이터에 대한 설명 |
| PowerPoint(ppt, pptx) 및 PDF 형식으로 제출 | 최종 결과 도출 과정 및 결과 해석 |
| ※ 보고서 작성에 사용한 폰트 파일 첨부 | |

- **(필수) 프레젠테이션 스크립트 파일 제출** (최종 분석보고서 제출 시 함께 제출)

- 시각화 파일
  - 시각화 결과는 QGIS를 작성하여 제출
  - QGIS 프로젝트 파일은 압축하여, 별도의 추가 설정 없이 실행 가능해야 함
  - ※ 실행 방법에 대한 설명 파일 추가 제출 가능
  - 제출된 모든 파일은 COMPAS 접속(인터넷) 환경에서 즉시 실행 가능해야 함

### 기타 참고 사항

- 수상작의 저작권은 수상자에게 귀속됩니다. 다만, 주최 기관은 수상작을 홍보 및 비영리 목적으로 자유롭게 활용(재작, 복제, 배포 등)할 수 있으며, 해당 목적 범위 내에서 일부 수정하여 사용할 수 있습니다.
